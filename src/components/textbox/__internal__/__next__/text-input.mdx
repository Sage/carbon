## Designer Notes

- Use placeholder text to give the user examples of data formats (e.g. AB123456C for a UK National Insurance number).
- You can disable a text input, but try to avoid this. If you need to, make it clear what the user needs to do in order to activate the text input.
- Use wider fields for longer data (e.g. an address line), and narrower fields for shorter data (e.g. a postcode), to give the user a clue about the data expected.

## Examples

### Default

A single-line text input. Provide a `label` and use `value` plus `onChange` for controlled behaviour. Supports common form-field props: `labelInline`, `labelWidth`, `inputWidth`, and validation.

<Canvas of={TextInputStories.Default} />

### Required

Set `required` to mark the field as mandatory. An asterisk appears in the label and the browser prevents form submission when the field is empty.

<Canvas of={TextInputStories.Required} />

### Placeholder

Set `placeholder` to display hint text inside the input when it has no value. Placeholder text disappears as soon as the user starts typing.

<Canvas of={TextInputStories.Placeholder} />

### With Input Hint

When the `inputHint` prop is passed, please use a full stop `.` at the end. This forces a pause
before any other announcements, this will help screen reader users understand the hint fully.

<Canvas of={TextInputStories.WithInputHint} />

### Disabled

Set `disabled` to prevent user interaction. The field is visually dimmed and excluded from the tab order.

<Canvas of={TextInputStories.Disabled} />

### Read-Only

Set `readOnly` to display the value as non-editable text within the field boundary. Unlike `disabled`, the value remains selectable and copyable.

<Canvas of={TextInputStories.ReadOnly} />

### Label Inline

Set `labelInline` to place the label to the left of the input rather than above it. Use `labelWidth` and `inputWidth` to adjust the column proportions.

<Canvas of={TextInputStories.LabelInline} />

### Sizes

Three sizes are available: `small`, `medium` (default), and `large`. Match the size to other form fields in the same layout.

<Canvas of={TextInputStories.Sizes} />

### Input Width

When `labelInline` is true, `inputWidth` controls the width of the input while maintaining visual balance with the label. 
When `labelInline` is false, `inputWidth` only affects the input width; the label width remains unchanged.

<Canvas of={TextInputStories.InputWidth} />

### Container Width

`containerWidth` controls the width of the entire component container which contains the input and the label. 

<Canvas of={TextInputStories.ContainerWidth} />
