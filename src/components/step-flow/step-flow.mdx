Provide a way to represent an end-to-end journey that a user can complete in one go.

This component has a specific start and end point, as well as showing the current step in the journey.

Use a step flow to help a user complete tasks in a specific order, based on their needs.

```js
  StepFlow,
  StepFlowTitle,
  type StepFlowHandle,
  type Steps
} from "carbon-react/lib/components/step-flow";
```

## Examples

### Default

To use this component, you must provide a `title`, `currentStep` and `totalSteps` props. The maximum number of steps that can be passed is `8`.

<Canvas of={StepFlowStories.Default} />

### Title as a Node

To allow a compositional approach, the `title` prop accepts a node. 
It is recommended to pass the `StepFlowTitle` component as a child of the node to ensure the correct styles and functionality are applied.

The `screenReaderOnlyTitle` prop can be used to pass a custom screen reader only title to the `StepFlowTitle`.
This is useful for providing screen reader users with necessary information or context which may only be available visually.

<Canvas of={StepFlowStories.TitleNode} />

### Title Variant

By default, the title is rendered as an `h1`, but this can be changed to `h2` by passing the `titleVariant` prop. 
The `StepFlowTitle` component can also be used to set the title variant.

<Canvas of={StepFlowStories.TitleVariant} />

### With Progress Indicator

You can set the `showProgressIndicator` prop to `true` to render the progress indicator within the component.

<Canvas of={StepFlowStories.ShowProgressIndicator} />

### With Close Icon

You can set the `showCloseIcon` prop to `true` to render a close icon within the component. 
You can also use the `onDismiss` prop to pass in a function which is called when a user clicks on the close icon.

<Canvas of={StepFlowStories.ShowCloseIcon} />

### Example Implementation

Please see below an example implementation of how the component can be used within the `Dialog` component to follow the recommended DS pattern for this component.
Use `Button` to advance through the user journey and `Typography` to display the additional Step Heading and descriptions as required. 

When advancing through the journey, call the `focus()` method exposed by the `StepFlow` component's forwarded ref via the `StepFlowHandle` imperative handle.
It is **strongly recommended** to call this method when advancing through the journey to ensure that the `StepFlow` component is focused and the screen reader users are informed of the current step in the journey.

| Method Name | Description                                              |
| ----------- | -------------------------------------------------------- |
| `focus()`   | Programmatically focuses the root container of StepFlow. |
