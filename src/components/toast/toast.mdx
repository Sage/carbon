A deprecated notification component that renders a temporary message overlay anchored to the viewport. Supports multiple variants (`success`, `info`, `neutral`, `error`, `warning`, `notice`, `notification`), configurable horizontal and vertical alignment, auto-dismiss with a timeout, stacking via `targetPortalId`, and an optional dismiss button.

**Category:** Feedback

## Contents

## Quick Start

To use toast, import the `Toast` and pass the content as children.

```javascript
import Toast from "carbon-react/lib/components/toast";
```

## Examples

### Controlled

```javascript
const MyComponent = () => (
  <Toast
    variant="info"
    id="toast-quick-start"
    open={isOpen}
    onDismiss={onDismissClick}
  >
    {children}
  </Toast>
);
```

It can be used as a uncontrolled component.

### Uncontrolled

```javascript
const MyComponent = () => (
  <Toast variant="info" id="toast-quick-start">
    {children}
  </Toast>
);
```

### Info

Set `variant="info"` to display an informational toast with a blue colour scheme and information icon.

<Canvas of={ToastStories.Info} />

### Neutral

Set `variant="neutral"` for a grey, non-semantic toast suitable for general-purpose messages that don't carry a specific sentiment.

<Canvas of={ToastStories.Neutral} />

### Error

Set `variant="error"` to show a red error toast. Use to notify users of a failure that requires their attention.

<Canvas of={ToastStories.Error} />

### Warning

Set `variant="warning"` for an amber warning toast. Use to alert users to a condition they should be aware of.

<Canvas of={ToastStories.Warning} />

### Notice

When the `"notice"` `variant` is set, the Toast component is rendered at the bottom of the screen with alternative styling and animation.
The `maxWidth` prop will be ignored when this variant is set.
To render it at the top of the screen use the `alignY` prop set to `"top"`.

<Canvas of={ToastStories.Notice} />

### Notification

Set `variant="notification"` for a dark-themed notification toast, suitable for system-level messages or push-style alerts.

<Canvas of={ToastStories.Notification} />

### Custom max width

Set `maxWidth` to constrain the toast width using any valid CSS value, preventing it from stretching too wide on large screens.

<Canvas of={ToastStories.CustomMaxWidth} />

### Dismissible

Set `onDismiss` to render a close button on the toast, letting users dismiss it manually before any auto-close timeout.

<Canvas of={ToastStories.Dismissible} />

### Dismissible with timeout

Set `timeout` (in milliseconds) to auto-dismiss the toast after the specified duration. A progress bar indicates the remaining time.

<Canvas of={ToastStories.DismissibleWithTimeout} />

### Dismissible without autoFocus

Set `autoFocus={false}` to prevent the toast from stealing keyboard focus when it appears. Ensure you add an `aria-live` region elsewhere to announce the toast for screen reader users.

<Canvas of={ToastStories.DismissibleWithoutAutoFocus} />

### Grouped Delayed (stacking)

Toasts that share a `targetPortalId` stack together. This example adds a one-second delay before the second toast appears.

<Canvas of={ToastStories.StackedDelayed} />

### Grouped (stacking)

Use `targetPortalId` to group multiple toasts into the same portal so they stack vertically rather than overlapping.
