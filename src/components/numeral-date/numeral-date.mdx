# Numeral Date

For dates far from today, use this Numeral Date component. It is advised to use Three inputs consisting of day, month, and year.
For dates close to today, we advise the use of a standard datepicker. If you require this, please see the "Date Input" component.

### Validating date values

`NumeralDate` has an error state, which can be set via the `error` prop. However, `NumeralDate` also has a built-in feature to validate date values automatically, so you don't have to recreate this logic yourself.
This can be enabled via the `enableInternalError` prop. The rendered messages are displayed alongside the messages defined with the regular `error` prop.

This in-built validation feature uses translation keys for the invalid date messages, so you can modify them to suit your localisation needs. The keys are `numeralDate.validation.day`, `numeralDate.validation.month`, and `numeralDate.validation.year`.

**Note**: This component does not support isolated input validation. An error in any single field will cause the entire component - including valid fields - to display an error state.
